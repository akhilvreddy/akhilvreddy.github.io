Tooling

After a lot of trial, error, and late-night debugging, these are the tools, libraries, and frameworks I’ve come to rely on.

Development Environment

#

Machine I use a MacBook Pro 16” (M4, 64GB Unified Memory, 512GB SSD). When getting a laptop, I wanted a machine that could train some models locally if needed and have enough compute to play around with open source models on the laptop itself. I love the 64GB because it lets me run a training loop + Cursor + FastAPI + Docker + a bunch of browser tabs without my machine slowing down at all.

Editors & Terminal

#

Cursor (Pro) I used to be super against using Cursor because it was “robbing” me of learning ML properly and it felt like cutting corners (especially when it throws in a tensor .unsqueeze by itself and solves bugs I hadn’t fully debugged yet, which was scary at first). But after using it I realized that it’s the direction that the world is going in and not using it would actually put me behind. I actually really like switching between the agent and manual modes.

Other than that, I use the main VSCode extensions and have Black, Pylance, Jupyter, and Dev Containers for ML workflows.

Warp I also disliked Warp at first because I used to be in awe of engineers that memorized that most unique flags for random CLI tools. But just like Cursor, you either use it to supercharge you or risk falling behind. Warp’s async output, smart command history, and AI enhancements make you faster in every way.

tmux I don’t use it much when developing locally, but on remote servers, it’s non-negotiable. I’ve used tmux at work and when I was doing research to run long training jobs, monitor logs, and switch sessions across SSH.

Shell zsh + Oh My Zsh + custom aliases + Starship prompt = you never have to worry if you are running code from the wrong place again.

Machine Learning Stack

#

PyTorch Probably my most favorite framework of all time. Building it from scratch in grad school really taught me how revolutionary the dynamic computation graph is.

Hugging Face Transformers + Datasets For NLP, fine-tuning, and zero/one shot classification.

Weights & Biases Training runs, experiment tracking, visualizations.

scikit-learn Go to tool when I’m not doing deep learning.

XGBoost/LightGBM For kaggle mostly: tabular supremacy, interpretability, and performance.

Triton I haven’t written many custom GPU kernels yet, but when I do, Triton feels way more intuitive than raw CUDA.

LangChain Only when I’m prototyping LLM chaining tasks quickly.

MLOps & Deployment

#

Docker For reproducible training environments. I use CUDA-based images for GPU workloads.

FastAPI My go-to for serving models with lightweight REST APIs.

TorchServe For proper model versioning + production inference.

ONNX For cross-framework model export + edge deployment.

MLflow For model registry + pipeline tracking (used in multi-model workflows).

AWS (SageMaker, EC2, S3) For cloud training, especially heavy workloads.

My favorite stack is Docker + FastAPI + TorchServe to deploy model APIs with versioning, health checks, and autoscaling on EC2.

Data Tools (Cleaning and Analysis)

#

Pandas GOAT framework in my opinion once you understand the annoying square bracket notation - I like cleaning data and restructuring data with it.

matplotlib + seaborn + plotly Each one of these excels in its own direction but I wish there was a unified visualization API that covers the strengths of all three without constantly switching between syntax or config styles.

Switching gears a little bit.

Frontend

#

I don’t do much frontend development anymore but I did spend a whole co-op writing AngularJS and worked on the entire frontend for an MVP of a failed startup.

React Simple, re-usable, and I actually feel like LLMs dominate in writing really good React code. I like pairing React with Tailwind and Vite for fast prototyping.

Angular I used this at my co-op for building data-dense, read-heavy interfaces. It did the job well in that context, but I haven’t seen Angular outperform React in flexibility or dev experience (I’m kind of happy Angular is fading away).

Backend

#

FastAPI My go-to Python web framework for building async-ready APIs with built-in OpenAPI docs. Type hints + Pydantic gives clean contracts, fast development, and easy debugging. I’ve used FastAPI for everything from model serving and microservices to quick internal tools.

Uvicorn Fast ASGI server I use to run FastAPI apps. It handles async HTTP requests with minimal overhead.

Gunicorn I pair this with Uvicorn behind Nginx when deploying to production, it handles worker management and scaling.

TypeScript (Node.js + Express / tRPC) When building full-stack apps, I write backends in TypeScript for type-safe API contracts and tight integration with React frontends. I’ve used Express, tRPC, and Next.js API routes depending on the app and it’s scale.

Databases

#

PostgreSQL My default RDBMS for anything with structured data. I like how it has flexible schema, JSONB support, and just works at scale. I’ve used it for everything from user auth to logging model predictions. Once I found out that Railways has a free tier I’ve basically used this for every small project.

SQLite Also a really good lightweight DB for local testing, prototyping, and quick scripts when spinning up a full Postgres DB isn’t worth it.

MongoDB My favorite NoSQL database - I use it mostly when doing full stack app dev, not that much in ML workflows.

Redis (light usage) Manages caching, rate limiting, real-time features, and lightweight queuing. I don’t use it too much when developing by myself.

RabbitMQ / Kafka (light usage) Used for async task handling and stream processing in some ML workflows (haven’t touched it much outside of work).

CI/CD and Containerization

#

GitHub Actions Handles my full CI/CD pipeline — tests, linters, type checks, and deployments. It keeps workflows minimal but reliable and I like how well it works with github.

Docker I like to containerize everything — dev environments, ML models, APIs because if it runs in Docker, I know it’ll run anywhere. It also makes deployments smooth across local, EC2, or K8s.

Makefiles I use it to script repeatable build + test + deploy workflows.

Infra / Cloud#

AWS (EC2, S3, SageMaker) Default cloud stack for hosting APIs, storing data, and training models.

Nginx Reverse proxy for Dockerized FastAPI apps. It takes care of routing, SSL, and healthchecks easily.

Terraform (for long-term infra) Learned it at an internship and realized that once you understand it, it’ll help setting up reusable environments. However, it is definitely overkill in a lot of places.

Docker Compose Go-to for spinning up multi-service apps locally — DB + API + frontend with one command.
